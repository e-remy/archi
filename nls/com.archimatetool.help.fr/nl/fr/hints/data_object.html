Un objet de données applicatif est un ensemble d'informations cohérent, auto-contenu et utilisable dans le cadre d'un traitement automatisé.

Une fonction applicative opère sur des objets de données applicatives.

Un objet de données applicatif peut être communiqué via des interactions et utilisé ou produit par des services applicatifs.

Le nom d'un objet de données applicatif devrait de préférence être un substantif. Il désigne de préférence un ensemble d'informations utile, auto-contenu et bénéficiant d'une signification claire également pour le métier, et pas uniquement au niveau applicatif.

Catégorie :

Application. Structure passive.

Exemples :

Données de fichier client, Données de police d'assurance, Données de souscription d'assurance, Base de données client.

Relations types :

Accès - composant applicatif, fonction applicative, interaction applicative, service applicatif
Réalisation - objet métier, artefact
Utilisé par - service applicatif
