Un artefact est une partie physique de l'information qui est utilisée ou produite au cours du processus de développement logiciel, de déploiement ou d'opération d'un système.

Un artefact représente un élément concret dans le monde physique. Il est typiquement utilisé pour modéliser des produits logiciels tels que des fichiers sources, des exécutables, des scripts, des tables de base de données, des messages, des documents, des spécifications et des fichiers de modèles. Une instance (copie) d'un artefact peut être déployée sur un nœud.

Un composant applicatif peut être réalisé par un ou plusieurs artefacts.

Un objet de données applicatif peut être réalisé par un ou plusieurs artefacts.

Un nœud peut être affecté à un artefact (i.e., l'artefact est déployé sur ce nœud).

Le nom d'un artefact devrait de préférence être le nom du fichier qu'il représente, par exemple "order.jar". Un artefact peut être composé de plusieurs sous-artefacts.

Catégorie :

Technologie. Structure passive.

Exemples :

EJB de gestion des risques, fichier JAR, widget, plug-in.

Relations types :

Réalisation - composant applicatif, objet de données applicatif
Affectation - artefact
