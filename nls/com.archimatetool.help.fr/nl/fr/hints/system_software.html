Un logiciel système représente un environnement logiciel pour les composants et objets qui sont déployés dessus sous la forme d'artefacts.

Un logiciel système est un nœud spécialisé qui est utilisé pour modéliser l'environnement logiciel dans lequel les artefacts s'exécutent. Un logiciel système peut également être utilisé pour représenter un integriciel (middleware) de communication.

Habituellement, un logiciel système est combiné à une machine représentant l'environnement matériel pour former un nœud plus complet.

Un logiciel système peut être affecté à une machine.

Les artefacts peuvent être affectés (i.e. déployés) à un logiciel système.

Un nœud peut contenir des logiciels système.

Le nom d'un logiciel système devrait de préférence être un substantif se référant au type d'environnement d'exécution, ex : "serveur JEE".

Un logiciel système peut contenir d'autres logiciels système, par exemple un système d'exploitation hébergeant une base de données.

Catégorie :

Technique. Structure active.

Exemples :

SGBD, système DB2, gestion des queues de messages, système d'exploitation, seveur d'application JEE

Relations types :

Affectation - machine, artefact
