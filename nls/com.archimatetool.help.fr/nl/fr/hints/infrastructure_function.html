Une fonction d'infrastructure décrit le comportement interne d'un nœud.

Elle regroupe le comportement de l'infrastructure qui peut être exécuté par un nœud.

Pour l'utilisateur d'un nœud qui exécute une fonction d'infrastructure, cette fonction est invisible. Si son comportement est exposé extérieurement, c'est au travers d'un ou plusieurs services d'infrastructure. Une fonction d'infrastructure est une abstraction de la façon dont celle-ci est réalisée. Seul le comportement nécessaire est spécifié.

Une fonction d'infrastructure peut réaliser des services d'infrastructure. Les services d'infrastrucutre d'autres fonctions d'infrastructure peuvent être utilisés par une fonction d'infrastructure. Une fonction d'infrastructure peut accéder à des artefacts. Un nœud peut être affecté à une fonction d'infrastructure (ce qui signifie que le nœud exécute la fonction d'infrastructure).

Le nom d'une fonction d'infrastructure devrait de préférence être un substantif indiquant une action, ex : "Gérer les données".

Catégorie :

Technique. Comportement.

Exemples :

Fournir un accès aux données, Gérer les données.

Relations types :

Utilisation par - nœud
Réalisation - service d'infrastructure
Affectation - nœud
